Create a template

This page describes how to create a new SIMIO template from an existing ALMA measurement set. The example uses Elias 24, one of the bright DSHARP planet-forming disks.

Step 1: Get the measurement set of the observation you want to mimic

For this tutorial, we will create a template based on Elias 24, one of the bright DSHARP planet-forming disks (Huang et al. 2018). The original measurement set can be obtained from the DSHARP data release.

Download the continuum of Elias 24. This file contains the information from the observation, including antennas, visibilities, times, spectral windows, and weights.

Step 2: Create a folder for your template

Go to your SIMIO directory and open the templates/ folder. Create an empty folder with the name of your template. In this example, the template will be called Elias24.

Step 3: Create subfolders and scripts

Inside the new template folder, create three empty folders named images, msfiles, and uvtables. Also create two empty scripts called template_casa_handling.py and template_info.py.

In this example, the template is Elias 24, so the scripts are called Elias24_casa_handling.py and Elias24_info.py.

Move the measurement set downloaded in Step 1 into the msfiles/ folder and extract the measurement set from the .tgz file.

Step 4: Template information

The file template_info.py, in this case Elias24_info.py, contains the information needed to generate the images for this template. You can also copy the info file from another template and replace the numbers with those of your new source.

# Source Geometry, Elias 2-24 in Huang et al. (2018), DSHARP II

dRa = 0. # arcsec

dDec = 0. # arcsec

inc = 29. # degrees

pa = 45.7 # degrees

geom_params = [dRa, dDec, inc, pa]

# Distance

parallax = 7.180888047224439 # GAIA EDR3 mas

dist = 1000. / parallax # pc

# Sky info

temp_center_ra = '16h26m24.0782s' # J2000

temp_center_dec = '-24d16m13.884s' # J2000

temp_semimaj = 1.15 # arcsec

# Clean info

temp_robust = 0.0 # From Andrews et al. (2021)

temp_cellsize = '0.003arcsec' # DSHARP standard

temp_imsize = 2000 # Big enough for noise estimation

Source geometry: The spatial offset, inclination, and position angle are given in degrees and arcseconds. For Elias 24 we set dRa and dDec to zero because the phase center will be corrected later.

Distance: The distance is estimated from the Gaia EDR3 parallax. The SIMBAD name of the source is Elias 2-24.

Sky information: The coordinates should correspond to the source center in J2000. Here, the coordinate is chosen from the peak-brightness pixel. The parameter temp_semimaj is the radial extent of the source in arcseconds and is used to create the CLEAN mask.

Clean information: These parameters are used to generate the template images. The robust parameter controls the weighting of the baselines. The cell size should be around seven to ten times smaller than the beam size, and the image size should be large enough to estimate the noise level with an annulus around the source.

Step 5: Prepare the measurement set in CASA

Open CASA from the template folder, for example from /home/path_to_simio/simio/templates/Elias24/. From this location, call the CASA analysis utilities and the SIMIO scripts needed to extract uv tables and generate images.

# Append the au functions from other directory.

sys.path.append('../../codes/analysis_scripts/')

import analysisUtils as au

# Import uv-handling functions

execfile('../../codes/simio_ms2ascii.py')

# Import imaging functions

execfile('../../codes/simio_clean.py')

# Info

prefix = 'Elias24'

ms_cont = 'msfiles/'+prefix+'_avg.ms'

listobs('msfiles/Elias24_continuum.ms')

Step 5.1: Open CASA and inspect the original measurement set

Set the name of the template as prefix, create the name for the averaged template measurement set, and execute listobs on the downloaded measurement set.

Step 5.2: Check the measurement set

The listobs output shows the spectral windows in the measurement set. For Elias 24, the visibilities are stored in 16 spectral windows. We will average each spectral window to one channel, and average in time to reduce the size of the template.

Step 5.3: Split and average

Remove previous auxiliary files, split the original measurement set into three auxiliary measurement sets, and average the spectral windows according to their original number of channels.

# Remove previous auxiliary files

os.system('rm -rf msfiles/Elias24_avg.ms')

os.system('rm -rf msfiles/Elias24_avg_aux1.ms')

os.system('rm -rf msfiles/Elias24_avg_aux2.ms')

os.system('rm -rf msfiles/Elias24_avg_aux3.ms')

# Split the data to average and combine

split(vis='msfiles/Elias24_continuum.ms', \
      outputvis='msfiles/Elias24_avg_aux1.ms', \
      timebin='30s', \
      spw='0,1,3,4,5,6', \
      width=8, \
      datacolumn='data', \
      intent='OBSERVE_TARGET#ON_SOURCE')

split(vis='msfiles/Elias24_continuum.ms', \
      outputvis='msfiles/Elias24_avg_aux2.ms', \
      timebin='30s', \
      spw='11,15', \
      datacolumn='data', \
      intent='OBSERVE_TARGET#ON_SOURCE')

split(vis='msfiles/Elias24_continuum.ms', \
      outputvis='msfiles/Elias24_avg_aux3.ms', \
      timebin='30s', \
      spw='2,7,8,9,10,12,13,14', \
      width=16, \
      datacolumn='data', \
      intent='OBSERVE_TARGET#ON_SOURCE')

This step can also be done with mstransform.

Step 5.4: Concatenate

Concatenate the auxiliary measurement sets into one averaged measurement set. To save disk space, remove the intermediate files afterwards.

# Join together

concat(vis=['msfiles/Elias24_avg_aux1.ms', \
            'msfiles/Elias24_avg_aux2.ms', \
            'msfiles/Elias24_avg_aux3.ms'], \
       concatvis='msfiles/Elias24_avg.ms', \
       dirtol='0.1arcsec', copypointing=False)

# Remove auxiliary files

os.system('rm -rf msfiles/Elias24_avg_aux1.ms')

os.system('rm -rf msfiles/Elias24_avg_aux2.ms')

os.system('rm -rf msfiles/Elias24_avg_aux3.ms')

# Check averaged Elias24

listobs('msfiles/Elias24_avg.ms')

After checking the new file with listobs, each spectral window should have only one channel.

Step 5.5: Shift the phase center

The phase center of this observation is not exactly at the center of the disk. We move the phase center of the visibilities to match the location of the peak emission of the disk. This step is optional for other templates.

# Move phase center

fixvis(vis=ms_cont, \
       outputvis=ms_cont, \
       phasecenter='J2000 16h26m24.0782s -24d16m13.884s')

Step 5.6: Create a mask for your template

Using the same information from the template info script, create a source mask and a residual mask. These masks use CASA region syntax.

# Mask geometry

inc, pa = 29., 45.7 # Huang et al. (2018), DSHARP II

pa_mask = str(pa)

# Mask semimajor axis size

mask_semimaj = 1.15

semimajor = str(mask_semimaj)+'arcsec'

semiminor = str(mask_semimaj*np.cos(np.deg2rad(inc)))+'arcsec'

# Mask center

center_ra = '16h26m24.0782s'

center_dec = '-24d16m13.884s'

center = center_ra + ' ' + center_dec

# Create masks

mask_LB = 'ellipse[[%s, %s], [%s, %s], %sdeg]' % \
          (center_ra, center_dec, semimajor, semiminor, pa_mask)

res_mask_LB = 'annulus[[%s, %s], [%s, %s]]' % \
              (center_ra, center_dec, '2.5arcsec', '3.5arcsec')

Step 6: Image your template

Step 6.1: Run CLEAN

Use tclean_wrapper to image the averaged measurement set. You can set the CLEAN parameters, but the cell size, image size, and robust value should match the values in the template information script.

The smallscalebias parameter makes CLEAN prefer larger scales before point-source components. The values gain=0.05 and cyclefactor=1.75 make the iterations more conservative. In this example, we clean down to four times the noise level so the JvM correction can be applied afterwards.

# Clean

tclean_wrapper(vis='msfiles/'+prefix+'_avg.ms', \
               imagename='images/'+prefix+'_im', \
               imsize=2000, \
               cellsize='0.003arcsec', \
               mask=mask_LB, \
               scales=[0, 3, 8, 13], \
               robust=0.0, \
               gain=0.05, \
               smallscalebias=0.32, \
               cyclefactor=1.75, \
               threshold=str(4*2.05e-02)+'mJy', \
               savemodel='modelcolumn', \
               niter=10000, \
               interactive=True)

Step 6.2: CLEAN interactively

The interactive CLEAN viewer should look like this, with the mask already set. Press the green arrow iteratively, or press the blue arrow and wait until the cleaning is finished. This step can take a long time.

Step 6.3: Estimate SNR and apply the JvM correction

After generating the images, execute the estimate_SNR function and apply the JvM correction to the newly created images. This gives the angular resolution and noise information for the template images.

Check the JvM-corrected image with the CASA viewer. In this example, the brightness scale has been adjusted.

Step 6.4: Export your images to FITS files

Use exportfits to export the newly created images. This example saves the JvM-corrected image, the non-JvM-corrected image, the CLEAN model, the residuals, and the PSF.

exportfits(imagename='images/'+prefix+'_im.image', \
           fitsimage='images/'+prefix+'_im_noJvM.fits', \
           history=False, overwrite=True)

exportfits(imagename='images/'+prefix+'_im.JvMcorr.image', \
           fitsimage='images/'+prefix+'_im.fits', \
           history=False, overwrite=True)

exportfits(imagename='images/'+prefix+'_im.model', \
           fitsimage='images/'+prefix+'_im_model.fits', \
           history=False, overwrite=True)

exportfits(imagename='images/'+prefix+'_im.residual', \
           fitsimage='images/'+prefix+'_im_residual.fits', \
           history=False, overwrite=True)

exportfits(imagename='images/'+prefix+'_im.psf', \
           fitsimage='images/'+prefix+'_im_psf.fits', \
           history=False, overwrite=True)

Step 7: Write the uv tables in txt format

Execute the following code to extract the uv table from the measurement set. The only line you should need to change is the name of the template. The uv table size will be printed after concatenation.

# Initialize weight_spectrum column

initweights(vis=ms_cont, wtmode='weight', dowtsp=True)

# Extract spw info

os.system('rm -rf uvtables/'+prefix+'_spws*')

ms.open(ms_cont)

spw_info = ms.getspectralwindowinfo()

np.save('uvtables/'+prefix+'_spws', spw_info.keys())

ms.close()

os.system('rm -rf msfiles/'+prefix+'_cont_spw*')

os.system('rm -rf uvtables/'+prefix+'_cont_spw*')

for i in spw_info.keys():

    # Names

    ms_file = 'msfiles/' + prefix + '_cont_spw'+str(i)+'.ms'

    ascii_file = 'uvtables/' + prefix + '_cont_spw'+str(i)+'.txt'

    # Delete any previous spw file

    os.system('rm -rf ' + ms_file)

    os.system('rm -rf ' + ascii_file)

    # Split

    split(vis=ms_cont, outputvis=ms_file, \
          keepflags=False, datacolumn='data', spw=i)

    ms_to_ascii(ms_file, ascii_file, with_flags=True)

    # Delete temporal msfile

    os.system('rm -rf ' + ms_file)

# Combine uvtable from each spw into a single file

os.system('rm -rf uvtables/'+prefix+'_uvtable.txt')

os.system('ls -1tr uvtables/'+prefix+'_cont_spw*.txt | xargs cat >> uvtables/'+prefix+'_uvtable.txt')

# Check weight

os.system('du -h uvtables/'+prefix+'_uvtable.txt')

# Delete previous uvtables

for i in spw_info.keys():

    ascii_file = 'uvtables/' + prefix + '_cont_spw'+str(i)+'.txt'

    os.system('rm -rf ' + ascii_file)

Example output:

# 93M uvtables/Elias24_uvtable.txt

Step 8: That's it

Your template is ready to be used as a SIMIO template. The template images are stored in the images/ folder, the averaged measurement set is in msfiles/, and the visibilities are in the uvtables/ folder in txt format.

You can now delete the .log and .last files created by CASA.
